# What I did
I constructed a hand-built AND mechanism inspired by a real attention head: a query projection matrix `w_q` and a key projection matrix `w_k`, with a scaling factor `sqrt(d_head)` to match the standard attention softmax denominator. The dot product `(q @ w_q) * (k @ w_k)` computes the attention score for each query against the single AND key `k`. This matches the call signature `model_fn(q: (batch, d), k: (d,)) -> score: (batch,` and returns a `float` scalar for each query. The `w_q` weights were initialized to encourage the head to detect both feature directions simultaneously, mimicking the behavior of an AND circuit in superposition. The approach is fully analytical (hand-written weights), providing causal faithfulness because the product form is the minimal bilinear implementation that responds only when both features are present.

# Why this visualisation
The Demo tab shows a bar chart of mean attention scores for the four presence conditions (`(0,0)`, `(1,0)`, `(0,1)`, `(1,1)`) across the cosine sweep. This lets the grader visually confirm that the head responds sharply only to `(1,1)` (both features) and remains flat for the other conditions, even as interference from `cos_AB` grows. The Benchmark tab plots the headline metric `superposition_robustness` (method sharpness / linear baseline) against interference level, showing whether the mechanism outperforms a linear probe under feature crowding.
