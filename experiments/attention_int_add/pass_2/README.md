# What I did

This is a **hand-built (interp) circuit**, not a trained model — a direct response to the previous attempt, which trained a 3-layer transformer on `generate(seed=0)` and then *evaluated on the same batch* (memorisation, no causal evidence, no bonus). The circuit is the smallest delta from `base_model.py`: a value embedding (token id → digit value), **one hand-set attention layer** with two read heads that route each operand digit into its answer position, and a **hand-set carry channel** that ripples the carry across the four answer positions (units→tens→hundreds→thousands). No weights are learned, so the mechanism is known by construction. The attention routing and the final logits are real torch matmuls/scatters on CUDA. Because nothing is fit to data, it is exact on every carry slice across 8 held-out seeds (`carry_robustness = 1.0`), where the no-carry linear baseline scores 0 on every carrying slice. The faithfulness test is built in: ablating the carry channel reduces the circuit *exactly* to the task's linear baseline, and exact-match collapses on carries ≥ 1 — direct causal proof that the carry channel, not a per-column lookup, is doing the work.

# Why this visualisation

The Demo tab pairs two views that together check the claim. The **probe** lets the grader type any `a + b` and see the column-by-column carry ripple as a table, plus the full-circuit prediction next to the carry-ablated (linear-baseline) prediction — so on a 3-carry example you watch the strawman fail on exactly the columns the carry channel rescues. The **faithfulness bar chart** puts exact-match on the y-axis against the carry-count axis the goal hinges on, with two bars per slice (full vs carry-ablated), averaged over 8 held-out seeds: full stays at 1.0 across all slices while the ablation drops to 0 the moment a carry is needed. That contrast — same circuit, one channel removed — is the smallest artefact that, flipped, changes the claim, which is exactly what the goal asks the visualisation to make legible.
