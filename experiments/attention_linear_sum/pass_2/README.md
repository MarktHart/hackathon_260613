*What I did*  
I built a single attention head with hand-tuned weights that reads two scalar features embedded at position 0 and 1, a coefficient token (α,β) at position 2, and outputs the linear combination α·x₁ + β·x₂ at every target position. The model is a delta from the baseline head architecture: I added the same parameters but initialised them explicitly to a known working pattern, and I omitted the MLP layer to isolate attention-only computation. The model function `model_fn` converts the NumPy batch to a PyTorch residual tensor, runs the head on the CPU, and returns the NumPy (B,5) output expected by `task.evaluate`.

*Why this visualisation*  
The Gradio app shows two panels: in **Demo**, a slider pair lets you pick α and β, and a bar chart of per-coefficient R² across the 24 pairs lets you see whether performance degrades at extreme coefficients. In **Benchmark**, the built-in leaderboard plots the sweep across all attempts, turning the headline metric (canonical α=β=1 R²) and robustness score (min R² / max R²) into a comparative line chart. This keeps the question visible — uniform high R² across sweeping coefficients — without cluttering with raw weights or per-token attention matrices.
