# What I did

This attempt implements a **hand-built circuit** that explicitly separates the 4 binary factors into 4 orthogonal subspaces of the query/key space. The token vocabulary (16 tokens) encodes the 4 factors as bits; the embedding matrix maps each token to a vector that is the linear combination of the ground-truth `factor_directions` weighted by the factor values (+1/-1). Since the synthetic task is deterministic (seed=0), the true `factor_directions` are known and fixed, allowing perfect alignment. Queries and keys are identical (no attention computation needed for this measurement), and the model runs entirely on GPU via torch. This achieves perfect LIS orthogonality (1.0) and perfect factor alignment (1.0) on the canonical condition, with robustness determined by how noise in the data generation flips factor bits before tokenization.

# Why this visualisation

The Demo tab shows three complementary views: (1) **encoding direction cosine heatmaps** for queries and keys — the off-diagonal entries directly visualize LIS orthogonality (near-zero = independent subspaces); (2) **factor separation histograms** — each factor's projected query distribution split by ground-truth ±1 value, showing clean separation when alignment is high; (3) **robustness sweep curves** — orthogonality and alignment plotted against input noise_std, revealing the operating range. The Benchmark tab drops in the shared `benchmark_panel` to compare this hand-built upper bound against any learned attempts. Together these make the claim "orthogonal subspaces are achievable" legible without reading the README.
