# What I did

**Attempt type: hand_built** (no learning, no trained model — a circuit written out by hand). I build a single real attention head, from the task's own random `W_Q, W_K`, that computes `A AND NOT B` by genuinely reading features out of a **superposed** residual stream. The features enter only through a superposed write at the query position, `x_q = feat_A·e_A + feat_B·e_B + p_bias`, using the task's unit directions `e_A, e_B` at cosine `theta`. The head's one hand-set vector is the A-token embedding `x_A`, solved so that *through the real QK matrix* `M = W_Q W_Kᵀ` the dot products satisfy `e_Aᵀ M x_A = +12`, `e_Bᵀ M x_A = −18`, `p_biasᵀ M x_A = −4`, making the query→A logit exactly `12·feat_A − 18·feat_B − 4`. Solving those three constraints *is* the dual (reciprocal) basis of `{e_A, e_B}` — it implicitly inverts the Gram matrix `[[1,cos],[cos,1]]` to cancel the cross-talk superposition injects, so the dual basis is not an add-on but what solving for a clean NOT forces. This fixes pass_5's flaw: features are never read as oracle labels and keys are never injected per sequence — everything flows through `e_A/e_B` and the real head geometry, on the GPU.

**Baseline / ablation / operating range.** Beyond the framework's linear baseline (no-`B` head, stuck at chance), I include the load-bearing ablation: a *naive* head that reads with the raw `e_A/e_B` directions instead of the dual basis. Its logit degrades to `(12−18cos)·feat_A + (12cos−18)·feat_B − 4`; at `cos=0` it equals the corrected head, but its suppression gap collapses from `~1.0` to `~0` by `cos=0.8` — the query stops attending to A at all. I also record the **norm cost** `||x_A||`, which grows as `cos→1`: separating overlapping directions is exactly what costs read-out norm, so this quantifies *why* superposition is hard rather than hiding behind a flat number.

# Why this visualisation

Three panels, each the smallest artefact that would flip the claim if zeroed:

1. **Suppression gap vs cos** (continuous, not the lenient AUC) for corrected vs naive vs the chance baseline. This is the right y-axis: the NOT-sharpness *AUC* only checks ordering and stays ~1.0 even for the broken naive head, but the gap reveals that the naive query barely attends to A under superposition. The corrected curve staying flat while the naive curve collapses is the whole result, made visible.
2. **Norm cost `||x_A||` vs cos** — the mechanism's price. A rising curve says the dual-basis read-out is doing real, increasingly expensive work as `e_A` and `e_B` align, which is the genuine engagement with the superposition axis the goal asks about.
3. **query→A attention bars by (A,B) condition at `cos=0.8`**, corrected vs naive — concrete proof at the hardest slice that the corrected head attends only on `A1B0` while the naive head has lost the signal. Together they let a human verify the NOT survives superposition *and* see exactly which circuit element earns that survival.
