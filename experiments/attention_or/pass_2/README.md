## What I did

I built a hand-set **max-over-detectors attention** mechanism that directly implements logical OR over two feature directions, anchored on the `base_model.py`-style attention kernel. The attempt is a small delta from the synthetic construction: I compute basic scaled dot-product logits with QKᵀ, then apply an *additive* bonus only at the two signal key positions when the query is a balanced superposition `q_AB = (q_A + q_B) / ‖q_A + q_B‖`. That bonus turns the effective query into `q_eff = q_AB / ‖q_AB‖ + β·k`, where `k = (1,1,0,...,0)` fires only at the two signal columns, so the combined query attends to *both* keys (like OR) while individual queries retain pure base logits (like AND/linear). I do not train any parameters; the bonus magnitude `β` is taken from the canonical cos=0.0 batch (`batch.q_A @ batch.q_B`). This is a **hand-coded max-pooling trick expressed natively as attention**, not as a hard max over precomputed directions.

## Why this visualisation

The Demo tab shows two figures that together answer the goal question: (1) **Or sharpness vs cos(q_A, q_B)** compares the hand-built OR mechanism (blue) to the linear superposition reference (orange). Sharpness = `min(s_AB_at_A, s_AB_at_B) / max(s_A_at_A, s_B_at_B)`; at the canonical anchor (cos=0) the OR curve should reach 1.0 (perfect max), while the linear reference should drop below it, showing the "lift" from true OR. (2) **Noise leakage from q_AB** plots the mass that leaks to the non-signal columns as a fraction of the total signal mass; a low leakage line shows that the combined query's attention mass stays concentrated at the two intended keys, not bleeding into noise. These panels together verify both *separation* (sharpness vs the linear baseline) and *focus* (low leakage), and show how the mechanism behaves across the feature-direction overlap sweep.
