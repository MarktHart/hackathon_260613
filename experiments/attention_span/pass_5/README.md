**What I did**

This is a **hand_built** attempt (no training — every weight is set by hand in `build_head`). It is the smallest delta from `base_model.py`: a single attention head with a token embedding and Q/K projections, scaled-dot-product softmax, and **no MLP**. The hypothesis is that the *effective span* of attention is bounded by **content addressing, not distance**: I set the embedding so the query token `8888` and needle token `9999` both project (via `W_Q`/`W_K`) onto a shared "retrieve" direction, so the query→needle logit is a constant (`score_scale=12`) regardless of how far apart they sit. The head therefore holds ~0.997 attention on the target flat across the whole sweep (d = 1…256, >2 orders of magnitude), giving a robustness ratio (long/short) ≈ 1.0 and an AUC of 0.997 vs the uniform baseline 1/512 ≈ 0.002. I include two contrasts computed in the same forward pass: a **positional-only strawman** (ALiBi distance penalty, content zeroed) whose attention decays from 0.046 at d=1 to ~0 by d=256 (AUC 0.022, a genuinely *finite* span), and a **causal ablation** that zeros the needle key direction in `W_K` and collapses attention exactly to uniform (AUC 0.002) — proving the span is produced by the Q/K alignment and nothing else. All matmuls (embedding, Q/K projection, query-row scores, softmax) run in torch on CUDA.

**Why this visualisation**

The Demo plots mean query→target attention (y) against `log2(distance)` (x), the natural axis for a geometric sweep that spans 1→256 in 9 points, with the uniform baseline drawn as a reference line. Putting the content head, the positional strawman, and the ablation on one chart makes the headline claim legible at a glance: one flat line near the top (content span is distance-invariant), one decaying line (positional span is finite), and one line pinned on the baseline (ablation kills the mechanism). The interactive ALiBi-slope slider turns "span" into something you can move — raising the slope visibly shrinks the last distance whose attention still beats uniform — which is exactly the quantity the goal asks about. The Benchmark tab drops in the shared `benchmark_panel` so this attempt's AUC and lift-over-baseline sit alongside every other attempt.
