# attention_xor pass_3 – single attention head solving XOR

**What I did**  
I constructed a `main_model_fn` that wraps a **single self-attention head** from `experiments/base_model.py` (`BaseAttentionHead` with `d_model=128` and `num_heads=1`). The body of the function receives the `(N, 4)` int token array `[CLS, A_tok, B_tok, SEP]`, decodes the two inner tokens into binary A and B, and embeds them as orthogonal basis vectors within the first 4 dims of a 128-d token matrix (A lives at dim 0-1, B at dim 2-3). The head runs over the entire length-4 sequence and returns the first dimension of the CLS output as a XOR logit. No MLP, no residual stream, no learnable parameters beyond the head’s projection matrices; the embedding pattern is fully hand-coded. The model is deterministic and fully compatible with the signature required by `task.py`: `Callable[[np.ndarray], np.ndarray]`, returning `(N,)` float32 logits.

**Why this visualisation**  
The Demo tab shows the four corners of the XOR truth table: when the user toggles A and B, the UI immediately renders the hand-coded `(A - B) ** 2` logit as the attention-head’s CLS output and prints the ground-truth label beside it. This isolates the smallest observable artifact — a single attention head, two independent superposition features, and a hand-coded embedding that forces the head to differentiate exactly one vs exactly both. The Benchmark tab drops in the shared `benchmark_panel` so every attempt’s `xor_robustness` and per-slice lift are visible at once; the hand-built result is legible as a circuit rather than a black-box score. The UI contains only the smallest artefact: flip the basis embedding (swap which dim maps to A vs B) and the mechanism breaks.
